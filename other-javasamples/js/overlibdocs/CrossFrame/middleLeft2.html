Frame [2]

This file set has tests of the overlibmws across frame popup support via its overlibmws_crossframe.js and overlibmws_regCore.js plugin modules.

This link is to the 1st file in this frame. Click the link while a popup is being displayed in this frame (via the lower links in the upper-right frame, see below), to see if your browser on your platform handles the navigation without complaints. Also try this via links in the popups, and in the lower-right frame.

This file set has a series of frames. Each frame is labeled in the upper left hand corner. The frame notation also indicates whether the frame is nested or not. By nested, one means that the frameset loaded a document that itself was a frameset document. This frame (Frame [2]) is the third frame defined in the original frameset document. The frame in the middle, upper right (Frame [3][0][1]) represents a third order of nesting.

The links in the Frame [3][0][1] will cause a series of STICKY popups (or a non-sticky BUBBLE popup) to appear in their respective frames, in the upper left hand corner via REL-based positioning, or over the above link via REF-based positioning. The core and crossframe plugin modules include the frame reference to where a popup was originally loaded and this is reset each time a new popup is shown. If this page is scrolled, the popups with REL-based positioning should follow the scrolling.

You also can use frame references to access the same overlibmws module{s) in another frame, as this link and this link but not this link show.
